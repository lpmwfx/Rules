# MCP Session Management

> Sessions survive server restarts — transparent recovery, no client reconnect

---

## Session Lifecycle

Each `/mcp` POST creates or reuses a session. A session holds a McpServer + transport pair, tracked by `mcp-session-id` header.

```
Client POST /mcp (no header)     → new session, return mcp-session-id
Client POST /mcp (with header)   → reuse existing session
Client DELETE /mcp (with header)  → close session
```

RULE: Sessions stored in a Map<string, {server, transport}>
RULE: Session ID is a UUID generated by the transport

## Session Recovery

After server restart, clients send stale session IDs. Instead of returning an error, recover transparently:

1. Create new session with the **same** stale ID
2. Fake initialize + initialized handshake internally
3. Forward the real request

The client never notices the restart.

```typescript
async function recoverSession(staleId: string, req: Request): Promise<Response> {
  const transport = createSession(staleId);

  // Fake initialize
  const initReq = new Request(req.url, {
    method: "POST",
    headers: { "Content-Type": "application/json", "Accept": "application/json, text/event-stream" },
    body: JSON.stringify({
      jsonrpc: "2.0", id: "_recover_",
      method: "initialize",
      params: { protocolVersion: "2025-03-26", capabilities: {}, clientInfo: { name: "recovery", version: "0.1" } },
    }),
  });
  await transport.handleRequest(initReq);

  // Fake initialized notification
  const notifReq = new Request(req.url, {
    method: "POST",
    headers: { "Content-Type": "application/json", "Accept": "application/json, text/event-stream", "mcp-session-id": staleId },
    body: JSON.stringify({ jsonrpc: "2.0", method: "notifications/initialized" }),
  });
  await transport.handleRequest(notifReq);

  return transport.handleRequest(req);
}
```

RULE: Stale session IDs are recovered, not rejected
RULE: Recovery is transparent — client sees no error
RULE: Recovery creates a fresh server+transport pair with the same ID
BANNED: Returning 404 or error for stale sessions — always recover
